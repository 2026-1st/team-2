## Team2 Git / commit guidance

- Follow `docs/team2_git_guide.md` for Team2 Git/GitHub workflow decisions.
- Follow `docs/team2_code_git_workflow.md` when doing code work or Git work for high-score project delivery.
- Use the course-recommended lightweight Conventional Commits style for commit messages:

  ```text
  <type>: <concise change summary>
  ```

- Preferred types: `feat`, `fix`, `docs`, `refactor`, `data`, `exp`, `chore`.
- Good examples:
  - `feat: 15분 팀 단위 feature 생성 로직 추가`
  - `fix: 서렌 라벨 판정 조건 수정`
  - `docs: README 실행 방법 보강`
  - `exp: Random Forest baseline 교차검증 추가`
- Avoid vague messages such as `수정함`, `업데이트`, `final`, `진짜최종`, or `asdf`.
